## THE PROMPT

```markdown
ROLE
You are a senior frontend engineer specializing in cinematic, scroll-driven web 
experiences using Next.js 14, Framer Motion, and Lenis. You've shipped 
Awwwards-recognized sites. Your code is production-grade, performant, and follows 
React best practices.

CONTEXT
This codebase was generated by [PASS 1 MODEL — e.g. "Kimi K2" or "GPT-4V"] from 
a [REFERENCE SOURCE — e.g. "Pinterest reference image"] for [CLIENT/PROJECT NAME]. 
The model nailed the layout and visual hierarchy, but missed several scroll-driven 
motion behaviors and one critical structural issue. You're refactoring it into the 
production version.

The reference aesthetic is [3-5 ADJECTIVES — e.g. "editorial cinematic — Wes Anderson meets 
Studio Ghibli"]. Layered depth, intentional pacing, no corporate motion. Every animation 
should feel like it was art-directed, not bolted on.

You will solve four problems in this single pass, in priority order. Each fix builds 
on the previous one — do not work on them independently.

---

PROBLEM 1 — FIX THE HERO LAYERING (CRITICAL — DO THIS FIRST)

[If the hero has a z-index layering concept — e.g. typography sitting behind a landscape:]

The hero section has a structural bug. The [ELEMENT — e.g. "display typography"] should 
sit BEHIND the [ELEMENT — e.g. "landscape image"] — only [WHICH PART] should be visible 
[HOW — e.g. "above the mountain line"]. Currently, [WHAT'S WRONG].

Refactor the hero composition using a proper layered approach:

1. Background plane: [DESCRIPTION]. Position absolute, full viewport.
2. Mid-ground plane: [DESCRIPTION], with z-index BELOW [ELEMENT] but ABOVE [ELEMENT].
3. Foreground plane: [DESCRIPTION], highest z-index in the hero stack.

The cleanest implementation: [DESCRIBE THE APPROACH — split images, CSS mask, clip-path].

Verify visually: [DESCRIBE WHAT CORRECT LOOKS LIKE].

[If no z-index issue exists, replace with:]
The hero composition is structurally correct. Verify the z-index stack is intentional 
and move to Problem 2.

---

PROBLEM 2 — SCROLL-LINKED PARALLAX

Once the layering is correct, add scroll-linked parallax to the hero section using 
Framer Motion's useScroll and useTransform hooks.

Movement specifications:
- [ELEMENT — background]: translateY at 0.3x scroll speed (slowest — anchors the scene)
- [ELEMENT — headline/mid]: translateY at 0.5x scroll speed (mid-depth)
- [ELEMENT — if applicable]: translateX at 0.4x scroll speed
- [ELEMENT — focal point]: stays fixed, no transform (visual anchor)

Implementation requirements:
- Use useScroll with target ref on the hero section, offset: ["start start", "end start"]
- Use useTransform to map scrollYProgress to pixel translations
- Apply transforms via motion.div wrappers
- All transforms must use willChange: "transform" for GPU acceleration
- Easing: linear — parallax should feel mechanical and tied precisely to scroll position

---

PROBLEM 3 — STAGGER REVEAL

The [SECTION NAME — e.g. "About" or "Timeline"] section currently renders all items at 
full opacity simultaneously. This needs to be a viewport-triggered sequential reveal.

Refactor the component:

1. Wrap each item in its own motion.div
2. Use Framer Motion's useInView hook on each item with:
   - threshold: 0.3 (triggers when 30% is in view)
   - once: true (only animates the first time)
3. Animation states:
   - initial: { opacity: 0, y: 40 }
   - animate (when inView): { opacity: 1, y: 0 }
   - transition: { duration: 0.8, ease: [0.16, 1, 0.3, 1] }
4. Stagger: 200ms delay between items. Either:
   - Parent container with staggerChildren: 0.2, OR
   - Individual delay props: item 1 = 0ms, item 2 = 200ms, item 3 = 400ms

The sequence should feel intentional — each item arrives with purpose, not all at once.

---

PROBLEM 4 — PRODUCTION POLISH

Three final touches that elevate this from "AI demo" to "shippable product":

A) LENIS SMOOTH SCROLL
Wrap the entire app in a Lenis smooth scroll provider at the root layout level. 
Use @studio-freight/lenis. Configuration:
- duration: 1.2
- easing: t => Math.min(1, 1.001 - Math.pow(2, -10 * t))
- smooth: true
- smoothTouch: false (mobile keeps native scroll for performance)

Ensure Framer Motion's useScroll continues to work correctly with Lenis.
Clean up the Lenis instance on unmount.

B) CUSTOM CURSOR
Build a custom cursor component with two states:
- Default: 8px solid dot, follows mouse with smooth lerp (~0.15 lag)
- Hover: 32px outlined circle (1px border, no fill), triggered on any interactive element

Implementation:
- Position fixed, pointer-events: none
- useMousePosition hook with requestAnimationFrame
- mix-blend-mode: difference (visible on any background)
- cursor: none on body
- Hide entirely on mobile (window.innerWidth < 768)

C) CARD HOVER STATES
All interactive cards need:
- whileHover: { y: -8, scale: 1.02 }
- transition: { duration: 0.4, ease: [0.16, 1, 0.3, 1] }
- Subtle glow on hover: box-shadow with brand accent at 20% opacity
- Cursor changes to hover state on card entry

---

DELIVERABLES

1. Refactored hero section with correct layering
2. Updated hero with scroll-linked parallax
3. Refactored [SECTION] with sequential viewport-triggered reveals
4. Lenis smooth scroll wrapper at root layout
5. CustomCursor component integrated into root layout
6. Updated cards with hover states

CONSTRAINTS

- Do NOT rewrite components that are already working correctly. Surgical edits only.
- Do NOT break the existing visual layout. The reference is the source of truth.
- Do NOT add dependencies beyond Lenis. Everything else solves with what's installed.
- Do NOT use spring physics for scroll-linked transforms. Linear/cubic-bezier only.
- All animations must respect prefers-reduced-motion.

OUTPUT FORMAT

For each fix:
1. File path being modified
2. 2–3 sentence explanation of the change and why
3. Full updated code (not diffs)

Work through Problems 1 → 4 in strict order. Verify the previous one still works 
before moving to the next.
```
